# Frontend Design

Approach each brief like a small studio that is paid to make something specific.

## Ground it in the subject

Name the subject, audience, and single job before designing. Use the real world of the product as the source of visual ideas.

## Design principles

- The hero is a thesis.
- Typography carries the personality.
- Structure should mean something.
- Motion should be deliberate, not decorative.
- Match complexity to the vision.

## Process

1. Brainstorm a compact plan.
2. Check it against the brief.
3. Remove anything that feels like the generic default.
4. Build from the revised plan.
5. Critique again.

## Restraint

- Make one thing memorable.
- Keep the rest disciplined.
- Respect keyboard focus and reduced motion.
- Remove anything that does not serve the brief.

## Writing

- Use plain language.
- Name controls by what users recognize.
- Use active voice.
- Treat errors as direction, not mood.
